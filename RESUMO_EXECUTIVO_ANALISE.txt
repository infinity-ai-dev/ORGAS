🎯 O QUE É O PROJETO
═══════════════════════════════════════════════════════════════════════════════

Sistema inteligente de classificação e roteamento de documentos baseado em
LangGraph/LangChain que:

1. Classifica automaticamente documentos (fiscal, accounting, personal, support, generic)
2. Roteia para sub-grafo especializado
3. Extrai dados estruturados com 5 módulos reutilizáveis
4. Garante compliance (LGPD/GDPR)
5. Retorna JSON estruturado pronto para integração

FLUXO:
Documento → Classificação (padrão matching) → Validação → Roteamento →
Execução (5 módulos) → Review (HITL) → Consolidação → JSON

🔴 CRÍTICAS ENCONTRADAS (MÁXIMA PRIORIDADE)
═══════════════════════════════════════════════════════════════════════════════

⚠️  REVOGAR IMEDIATAMENTE! ⚠️

1. 🔴 EXPOSIÇÃO DE CHAVES DE API (SEC-001)
   ├─ Arquivo: .env, docker-compose.test.yml
   ├─ Chaves expostas:
   │  ├─ GOOGLE_API_KEY=AIzaSyA7ETCJavsGamOYua4ra6eN7NAt7IyqDEg
   │  ├─ GROK_API_KEY=<REDACTED>
   │  └─ ANTHROPIC_API_KEY=sk-ant-api03-...
   ├─ Impacto: Roubo de identidade, consumo não autorizado de APIs, custos!
   └─ Fix: Revogar em Google/OpenAI/xAI/Anthropic, usar secrets manager

2. 🔴 VULNERABILIDADE XSS em /console (SEC-002)
   ├─ Arquivo: main.py, linhas 207, 217, 220
   ├─ Código: output.innerHTML += '<div>...' + msg + '...</div>';
   ├─ Problema: Input do usuário sem sanitização
   ├─ Payload: <img src=x onerror="stealCookie()">
   ├─ Impacto: Roubo de cookies/tokens, redirecionamento malicioso
   └─ Fix: Usar textContent ou DOMPurify.sanitize()

3. 🔴 FALTA DE AUTENTICAÇÃO (SEC-003)
   ├─ Arquivo: main.py
   ├─ Endpoints afetados: /agent, /health, /
   ├─ Problema: SEM validação de credenciais
   ├─ Impacto: Qualquer pessoa pode enviar documentos, consumir APIs
   └─ Fix: Implementar HTTPBearer ou JWT
